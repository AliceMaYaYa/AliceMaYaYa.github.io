微信小程序调研总结

发表于 2017-05-05

去年九月份微信小程序处于内测阶段，那时还是名在校学生，正好自己暑期实习接触到嵌套在微信端的WebApp，好奇的研究一下微信小程序和WebApp及APP的不同，和实验室小伙伴也讨论过，正好现在工作的部门让调研微信小程序，那就直接上干货。



## 微信小程序是什么

### 文字层面上的小程序：

张小龙说:“小程序是一种不需要下载安装即可使用的应用，它实现了应用“触手可及”的梦想，用户扫一扫或者搜一下即可打开应用。也体现了“用完即走”的理念，用户不用关心是否安装太多应用的问题。应用将无处不在，随时可用，但又无需安装卸载。”

由于小程序是在微信内部使用，无疑可以统一Android、IOS、WP三大平台，实现了一套程序，在多处运行，相当于微信成为了一个系统架构，避开了原始的微信搭载环境。

### 技术层面上的小程序：

微信小程序 = XML + CSS + JS 的移动应用程序

微信小程序的开发技术上是使用一种类似 React Native 的框架来保证程序的原生性和稳定性。

## 小程序与原生APP，H5 webapp的区别？

### 小程序与原生APP之间的区别：

- 功能实现：小程序：限于微信平台提供的功能; App：可实现完整功能 ;
- 下载安装：小程序：通过微信(扫描二维码、搜索、分享)即可获得; App：从应用商店(App Store、应用汇等)下载安装;
- 开发周期：小程序：平均开发周期约2周; App：一款完善的双平台App平均的开发周期约2个月;
- 内存占用：小程序：无需安装，和微信共用内存使用，占用内存空间忽略不计; App：安装于手机内存，一直占用内存空间，太多的 App 可能会导致内存不足;
- 手机适配：小程序：一次开发，多终端适配; App：需适配各种主流手机，开发成本大;
- 产品发布：小程序：提交到微信公众平台审核，云推送; App：向十几个应用商店提交审核，且各应用商店所需资料不一样，非常繁琐;
- 推广难度：小程序：通过二维码、微信搜索、朋友分享等方式直接获得; App：需要用户主动下载十几M的安装包，在没有Wi-Fi的情况下推广困难;
- 消息推送：小程序：仅能回复模版消息，不允许主动给用户发送广告，良好的产品体验; App：频繁无用广告推送，骚扰用户造成没必要的困扰;

### 小程序与H5 webapp之间的区别:

- 运行环境：小程序的运行环境并非完整的浏览器，微信开发团队基于浏览器内核完全重构的一个内置解析器，针对小程序专门做了优化，配合自己定义的开发语言标准，提升了小程序的性能; H5 webapp：传统的H5的运行环境是浏览器，包括webview。
- 开发成本：小程序开发由微信团队提供了开发者工具，并且规范了开发标准，微信自定义的WXML、WXSS，WXML标签，作为一个开发者，只用专注写程序：当需要实现功能时，直接调用API就好; H5 webapp：从环境配置到前端框架(Angular、react、vue等)、模块管理工具(Webpack 、Browserify 等)、任务管理工具(Grunt、Gulp等)、浏览器兼容性等都需要开发人员掌握这些技能，开发成本较高。
- 获取系统级权限：小程序能获得更多的系统权限，比如网络通信状态、数据缓存能力等，这些系统级权限都可以和微信小程序无缝衔接; H5 webapp：H5的大多应用场景被定位在业务逻辑简单、功能单一的原因，页面交互不够流畅。
- 产品角度：小程序它提供了丰富的框架组件和API接口供开发者调用，具体包含：界面、视图、内容、按钮、导航等。在这些组件和接口的帮助下，运行在微信上的小程序基本保持和Native APP一样的流畅度，极大的提高了用户的体验; H5 webapp：微信底部会提供菜单作为功能引导入口，最终的结果还是引导用户到H5页面。

2、应用层级文件

  • app.js

app.js定义了全局的一些重要事件，比如程序启动时要做些什么，如何获取微信用户信息等等。App函数是一个全局函数，它的作用就是用来创建一个应用程序实例，每个应用程序都会有它的生命周期，因此一些重要的生命周期事件都会在这里定义。

  • app.json

app.json定义了程序有哪些页面，以及窗口标题及相关样式等。

  • app.wxss

app.wxss定义了全局样式，可以把它理解为一些所有页面公用的css样式。

小程序的优缺点？

优点：

1、适合快速场景化服务

2、可以快速验证客户需求

3、适合初创团队

4、试错成本低，需要较少时间和资金投入

5、可以迅速占领空白领域客户渠道

缺点：

1、小程序不能订阅收藏、不能分享朋友圈，只能通过好友分享转发和推荐等社交方式发现新的小程序，用完退出后下次再用找起来比较费劲。

２、小程序平台封闭性过高，大量原生系统服务无法使用。

３、不兼容某些WEB标准协议，不能跨浏览器使用。

４、不支持nmp，npm架构正是React Native所使用的。

５、组件单一，很多场景虽然可以使用，但功能仍旧单一。

总结

经过一个多星期的调研，可以用小程序做一个简单版的产品及功能性实验，由于小程序的开发成本、发布上线时间相对比较低。但小程序毕竟是微信端上的，如果想做出属于自己公司的产品还是需要独自做APP，拥有自己的产品线。
微信小程序自带的组件和API，在开发阶段不用考虑浏览器兼容性问题，运行环境下不会带来不可预估的Bug,因此在开发成本上小程序比以往的Web开发成本低很多。
