JSX简介

JSX是React的核心组成部分，是JavaScript和XML结合的一种格式，React创建JSX，利用HTML语法来创建虚拟DOM，当遇到<，JSX就当HTML解析，遇到 { 就当JavaScript解析。

  • 基于ECMAScript的一种新特性
  • 一种定义带属性树结构（DOM结构）的语法

JSX语法

大括号表示JavaScript，不要加引号，加引号就会被当成字符串。

1、元素命名

JSX的标签与函数名都是使用的驼峰命名。JSX里自定义组件标签名要求分别使用首字母大、小写来区分本地组件的类和 HTML 标签。render渲染时，会把大写的组件名定义为自定义组件，把小写的组件名定义为HTML自带的标签名进行渲染。
htmlFor和className，for和class为js的保留字，在书写for与class时需要修改为htmlFor何className，注意都是使用的驼峰命名。
所有React组件都可以采用自闭和的形式，包括div等。

4、JSX中绑定事件

JSX让事件直接绑定在元素上。

大括号仍然表示JavaScript表达式，返回一个事件处理函数。React并不是真正的绑定事件到每一个具体的元素上，而是采用事件代理的模式：在根节点document上为每种事件添加唯一的Listener，然后通过事件的target找到真实的触发元素。

5、JSX中使用样式

在JSX中使用样式和真实的样式很类似，通过style属性来定义，但和真实的DOM不同的是，属性值不能是字符串而必须为对象。

6、使用HTML标签

7、HTML转义

React会将所有要显示到DOM的字符串转义，防止XSS。如果所以如果 JSX 中含有转义后的实体字符比如 © (.) 最后显示到 DOM 中不会正确显示，因为React 自动把 © 中的特殊字符转义了。有几种解决办法：

  • 直接使用 UTF-8 字符
  • 使用对应字符的 Unicode 编码，查询编码
  • 使用数组组装
  • 直接插入原始的 HTML

8、自定义HTML属性

如果在JSX中使用的属性不存在于HTML的规范中，这个属性会被忽略，如果要使用自定义属性，可以使用data-前缀。
