React组件

发表于 2017-04-21

React核心内容之一就是组件，React用组件封装界面模块，整个界面就是一个大的组件，开发过程中就是不断优化和拆分界面组件、构造整个组件树的过程。

创建一个React组件

使用React.createClass方法创建组件，传入的参数是一个对象，对象必须定义一个render方法，render方法返回值为组件的渲染结构（有且只有一个组件实例），当返回值为 null/false 的时候，React 内部通过noscript标签替换。

2、无状态组件

除了可以通过 React.createClass 来创建组件以外，组件也可以通过一个普通的函数定义，函数的返回值为组件渲染的结果。

无状态组件能够优化性能，因为其内部不会维护状态，React内部不会有一个对应的组件实例，并且也没有生命周期hook。

将组件渲染到DOM中

建立好组件以后，将组件渲染到DOM中，需要两个步骤：

1、在HTML中定义一个元素，设置id属性

2、JSX中调用RenderDOM.render方法，第一个参数为组件，第二个为定义的DOM元素

组件属性Props

React组件可以传递属性给组件，通过 this.props 获取组件属性。React组件通过唯一的props接口避免了逻辑复杂性，让开发测试都更加容易。

属性相关的 API :

1、this.props: 获取属性值

2、getDefaultProps: 获取默认属性对象，会被调用一次，当组件类创建的时候就会被调用，返回值会被缓存起来，当组件被实例化过后如果传入的属性没有值，会返回默认属性值

3、this.props.children：子节点属性，表示组件的所有子节点，它的值有三种可能：如果当前组件没有子节点，就是 undefined ;如果有一个子节点，数据类型是 object ；如果有多个子节点，数据类型就是 array

4、propTypes: 属性类型检查

举个栗子：

属性传递的单向性

React单向数据流模式，数据的流动管道就是 props，流动的方向就是组件的层级自定向下的方向。所以一个组件是不能修改自身的属性的，组件的属性一定是通过父组件传递而来（或者默认属性）。

无状态组件属性

对于无状态组件，可以添加 .propTypes 和 .defaultProps 属性到函数上。

组件状态State

React 的一大创新，就是将组件看成是一个状态机，一开始有一个初始状态，然后用户互动，导致状态变化，从而触发重新渲染 UI 。

React中每个组件可以存储自己的当前状态，组件规范中定义了setState方法，每次调用都会更新组件的状态，触发render方法。

render方法是异步调用的，可以保证同步的多个setState方法只会触发一次render，有利于提高性能。state是组件的内部状态，除了初始化可能由props来决定，之后完全由组件自身去维护。

举个栗子：

Switch里面控制状态的 API 为：

1、this.state：组件的当前状态

2、getInitialState：获取组件的初始状态，在组件加载的时候会被调用一次，返回值赋予 this.state 作为初始值

3、this.setState：

（1）、组件状态改变时，可以通过 this.setState 修改状态

（2）、setState 方法支持按需修改，如 state 有两个字段，仅当 setState 传入的对象包含字段 key 才会修改属性

（3）、每次调用 setState 会导致重渲染调用 render 方法

（4）、直接修改 state 不会重渲染组件
